### Aufgabe 2: Unterschied IaaS vs. PaaS

**IaaS (Infrastructure as a Service):**
- Ich miete virtuelle Server
- Ich muss Betriebssystem und Software selbst installieren
- Beispiel: AWS EC2, Azure VMs

**PaaS (Platform as a Service):**
- Ich bekomme eine fertige Plattform
- Ich kümmere mich NUR um meinen Code
- Der Anbieter kümmert sich um den Rest
- Beispiel: Google App Engine, Azure App Service

## Teil-Challenge 2: Infrastructure as Code (IaC)

### Aufgabe 1: Automatisierung vs. Manuell

**Warum automatisch statt manuell?**

- Immer gleiche Konfiguration (keine Fehler durch Tippfehler)  
- Viel schneller (Minuten statt Stunden)  
- Bei Absturz einfach neu aufbauen  
- Alle Änderungen sind dokumentiert  

**Was ist IaC?**
Infrastructure as Code bedeutet: Ich beschreibe meine Server und Netzwerke in Dateien (wie Code). Dann kann ich sie automatisch erstellen lassen. Alles ist versioniert und wiederholbar.

### Aufgabe 2: Tools und Konzepte

**Docker:**
- Verpackt Programme in Container
- Läuft überall gleich (Laptop, Server, Cloud)

**Terraform:**
- Baut komplette Infrastruktur auf (Server, Netzwerke)
- Funktioniert bei AWS, Azure, Google usw.

**Kubernetes & Container-Orchestrierung:**

| Begriff | Bedeutung |
|---------|-----------|
| Container-Orchestrierung | Container automatisch verwalten |
| Kubernetes | Das bekannteste Tool dafür |
| Was es macht | Startet Container automatisch, skaliert bei Last, ersetzt kaputte Container |
