# Docker Networking & Volumes

## Networking

**Port-Weiterleitung:** Container-Ports auf Host verfügbar machen:
```
docker run --rm -d -p 3306:3306 mysql
```
```
docker run --rm -d -P mysql
```
**Dockerfile:**
```
EXPOSE 3306
```
**Netzwerkmodi:**
- **Bridge** (Standard): Container kommunizieren untereinander
- **None**: Kein Netzwerk
- **Host**: Container nutzt Host-Netzwerk

## Volumes

Daten persistent speichern (über Container-Lebenszyklus hinaus):
```
docker run -d -p 3306:3306 -v ~/data/mysql:/var/lib/mysql --name mysql mysql
```
**Eigenschaften:**
- Bleiben nach Container-Löschung erhalten
- Können zwischen Containern geteilt werden
- Werden nie automatisch gelöscht

---

## Praxis: MariaDB mit Docker

### Setup
```
docker run -d -p 3306:3306 -e MARIADB_ROOT_PASSWORD=secret --name mariadb mariadb
```
### Schritte

1. **Container starten** mit Port-Forwarding
2. **Mit HeidiSQL verbinden** (Host: localhost, Port: 3306, User: root, Pass: secret)
3. **Datenbank erstellen**
4. **Volume einrichten** für Persistenz:
```
docker run -d -p 3306:3306 -v mariadb_data:/var/lib/mysql mariadb
```
### Wichtig
- Security Group der EC2-Instanz muss Port 3306 freigeben
- Volumes bleiben nach docker rm erhalten

# Docker: Eigene Images bauen

## Was ist ein Dockerfile?

Ein Dockerfile ist eine Textdatei mit Anweisungen zum Bau eines eigenen OCI-Images.

**Wichtige Anweisungen:**

FROM        # Basis-Image (z.B. ubuntu:latest)
LABEL       # Metadaten (Autor, Version)
RUN         # Befehle ausführen (z.B. Pakete installieren)
COPY        # Dateien in Image kopieren
ADD         # Wie COPY, aber mit URL/Archivextraktion
CMD         # Standardbefehl beim Containerstart
ENTRYPOINT  # Hauptprozess des Containers
EXPOSE      # Port bekannt machen
ENV         # Umgebungsvariablen setzen
WORKDIR     # Arbeitsverzeichnis setzen
USER        # Benutzer wechseln
VOLUME      # Mountpoint deklarieren

# Docker: Eigenes Chuck Norris Image erstellen

## Verzeichnis vorbereiten

mkdir chucknorris-image
cd chucknorris-image

## Dockerfile erstellen

nano Dockerfile

## Dockerfile Inhalt (einfache Version mit curl)

FROM alpine:latest
RUN apk add --no-cache curl
CMD curl -s https://api.chucknorris.io/jokes/random | grep -o '"value":"[^"]*"' | cut -d '"' -f 4

## Image bauen

docker build -t chucknorris:1.0 .

## Container starten

docker run --rm chucknorris:1.0

![Docker](/images/chuck_proof.png)

## Dockerfile Inhalt (Version mit Endlosschleife - immer neue Witze)

FROM alpine:latest
RUN apk add --no-cache curl
CMD while true; do curl -s https://api.chucknorris.io/jokes/random | grep -o '"value":"[^"]*"' | cut -d '"' -f 4; sleep 5; done
